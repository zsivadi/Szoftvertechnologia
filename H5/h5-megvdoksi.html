Architektúra megvalósítása

Rétegek

A fejlesztés a H4 dokumentumban leírt többrétegű architektúrát követi:

  • UI réteg: konzolos interfész (Program.cs, GameController)
  • Logika réteg: CombatSystem, DialogueSystem, InventorySystem, SaveLoadService
  • Adatmodell: az ERD alapján megvalósított osztályok

Megvalósítás

A UI réteg felel a felhasználói interakciókért. A GameController vezérli a játék fő funkcióit, és meghívja a logikai modulokat.

A logikai réteg főbb elemei:

  • CombatSystem: harc, sebzés, győzelem/vesztés, loot-sorsolás
  • DialogueSystem: többlépéses párbeszédfolyamat
  • InventorySystem: tárgyak hozzáadása, használata, listázása
  • SaveLoadService: JSON alapú mentés és betöltés

Fő modulok és komponensek

GameController

A játék központi irányítója. Feladata a modulok vezérlése, a bemutató futtatása, és a logikai egységek összehangolása. 


    // Harc indítása
    CombatSystem combat = new CombatSystem(hero, enemy);
    combat.StartCombat();

CombatSystem

A harcrendszer a támadásokat, sebzéseket és az ellenfél halálát kezeli. Az ERD alapján kiterjesztett Enemy modellhez loot-sorsolási mechanizmus tartozik. 


    var reward = enemy.RollLoot();
    if (reward != null)
        inventory.AddItem(reward);

InventorySystem

A játékos tárgyait tároló és kezelő rendszer.

DialogueSystem

A rendszer támogatja a lépésenkénti párbeszéd megjelenítést.

SaveLoadService

A játékos teljes állapotát JSON formátumban menti és tölti vissza.

Tervezési minták

  • MVC: Model – Controller – Console View felosztás
  • Repository: későbbi bővítési lehetőség
  • Modularizált logika: önálló komponensek

Használt technológiák

  • C# (.NET 7)
  • Konzolos futtatási környezet
  • JSON (mentési formátum)
  • Git verziókezelés
